## 🍏 核心板块一：两数之和 (Two Sum)

### 1. 题目描述
给定一个整数数组 `nums` 和一个目标值 `target`，请在数组中找出和为目标值的两个整数，并返回它们的数组下标。（假设每种输入只会对应一个答案，且同一个元素不能重复出现）。
* **示例**：nums = [2, 7, 11, 15], target = 9 ➡️ **输出**：[0, 1]

### 2. 极简 Java 实现
```java
import java.util.HashMap;
import java.util.Map;

public class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>(); 
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i]; 
            if (map.containsKey(complement)) {
                return new int[] { map.get(complement), i };
            }
            map.put(nums[i], i);
        }
        return new int[] {};
    }
}
```

### 3. 核心知识点总结
* **核心工具**：哈希表 (`HashMap`)。它是一种高度结构化的“键值对 (Key-Value)”记事本。
* **终极特长**：【快】。只要你提供一个唯一的 Key，它能以 O(1) 的惊人速度直接揪出对应的 Value，而不需要像数组那样用 `for` 循环从头到尾苦苦搜寻。
* **本题策略**：Key 存数字，Value 存该数字在数组里对应的下标。利用空间换时间，将双重循环的 O(n²) 暴力解法优化至线性级别的 O(n)。
* **核心语法**：
  1. `map.put(K, V)`：往哈希表存入一条数据。
  2. `map.containsKey(K)`：以 O(1) 速度检查某个 Key 是否存在。
  3. `map.get(K)`：以 O(1) 速度根据 Key 拿回 Value。
* **复杂度分析**：时间复杂度 O(n)，空间复杂度 O(n)。

## 🍎 核心板块二：只出现一次的数字 (Single Number)

### 1. 题目描述
给你一个非空整数数组 `nums`，除了某个元素只出现了一次以外，其余每个元素均出现了两次。找出那个只出现了一次的元素。
* **硬性要求**：算法必须具备 O(n) 的线性时间复杂度，且不得使用任何额外空间。
* **示例**：nums = [4, 1, 2, 1, 2] ➡️ **输出**：4

### 2. 极简 Java 实现
```java
public class Solution {
    public int singleNumber(int[] nums) {
        int result = 0;
        for (int num : nums) {
            result ^= num; 
        }
        return result;
    }
}
```

### 3. 核心知识点总结
* **核心工具**：位运算之按位异或运算 (`^`)。
* **基本概念**：位运算是不经过任何包装、由计算机 CPU 直接在二进制位（0 和 1）上进行的底层运算，执行速度是硬件级别的快。
* **异或特长（消消乐特性）**：
  1. `x ^ x = 0` （相同为 0：任何数字和自己异或，直接自我毁灭变成 0）。
  2. `x ^ 0 = x` （任何数和 0 异或，结果还是自己）。
  3. 满足交换律与结合律（数字的顺序不影响消除结果）。
* **解题策略**：数组里那些出现了两次的数字，在连续异或的过程中会全部“成双成对地打架”化为 0，最后全场只剩下那个形单影只的、只出现一次的数字。
* **复杂度分析**：时间复杂度 O(n)，空间复杂度 O(1)（完美压榨空间，达到大厂零额外空间消耗的最高要求）。
