# Pinia总结

- **作用**:在多个组件的情况下，方便管理不同组件共用的数据
>data     => state  
>methods  => action  
>computer => getters  
>watch    => subscribe  

## 1.配置pinia环境
>*控制台安装:npm i pinia*
- main.ts文件写入
``` ts
//1.引入pinia
import {createPinia} from 'pinia'
//2.创建pinia
const pinia = createPinia()
//3.安装pinia
const app = createApp(App)
app.use(pinia)
app.mount('#app')
```
<br><br>

## 2.写sotre文件
- ### 需要创建一个文件夹/store/count.ts存放代码  (相当于全局封装，哪里需要哪里调)
```ts
import {defineStore} from 'pinia'
export const useCountStore = defineStore('count',{      //在定义名字的时候,尽量使用 use+功能 的格式
    state(){
        return{
            sum:6
        }
    }
})
```
- ### 目标组件接收

## 3.修改传递的参数
- ### (1)直接粗暴,直接在目表组件里面修改
```ts
countStore.sun +=1
```

- ### (2)批量修改数据
```js
//比如传递过来了很多数据，如a，b，c
countStore.$patch({
    a:111,
    b:222,
    c:333
})
```
>提示：使用普通方法countStore.a修改也行，但是使用$patch，Timeline只变更一次

- ### (3)使用actions修改(复杂,可选用)
```ts
state(){ return { sum:3 } }

actions:{                   //actions里面放的是一堆方法,用于相应组件中的"动作"
    add(value){
        if(sum<10){
            this.sum += value      //如果想修改自己提供的数据,使用**this.** ,this是当前创建的store
        }
    }
}
```

## 3.storeToRefs
- **注意**:在使用原版的ToRefs时,会让传递过去的store中的全部东西都包裹ref,  
这时候用storeToRefs就可以解决，它只关注store里面的数据，不会对方法进行ref包裹
```ts
import {storeToRefs} from 'pinia'
```

## 4.getters
- **作用**：对state里面的数据不满，想要修改可以使用getters

## 5.$subscribe
相比watch的优点：
1. 默认自带深度监听
2. 天生高性能：
擅长合并同类项！当遇到批量修改（$patch）时，普通的 watch 会触发几十次，它只触发一次，不容易卡
```ts
userStore.$subscribe(() => {
    console.log('触发了！'); 
});
```
