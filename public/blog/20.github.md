# GitHub 自动部署到服务器（CI/CD）完整流程总结

## 一、整体流程概览（非常重要）

整个自动部署流程可以理解为一条“自动流水线”：

1. 我在本地写代码（前端/后端）
2. push 代码到 GitHub 的 main 分支
3. GitHub 检测到 push 事件
4. GitHub Actions 自动触发 workflow（deploy.yml）
5. Actions 服务器通过 SSH 连接我的云服务器
6. 在服务器上执行一系列部署命令
7. 拉取最新代码（git pull origin main）
8. 安装依赖（npm install）
9. 重启后端服务（pm2 restart）
10. 新版本代码上线完成

---

## 二、核心组成部分

### 1. GitHub Repository（代码仓库）
作用：
- 存放前端 + 后端代码
- main 分支作为生产环境代码

---

### 2. GitHub Actions（自动化执行系统）
作用：
- 监听代码 push
- 自动执行部署脚本
- 相当于“云端自动执行机器”

**关键文件**：
.github/workflows/deploy.yml
>该文件夹及文件要放在项目根目录

---

### 3. GitHub Secrets（安全变量存储）
作用：
- 存放服务器敏感信息
- 防止泄露到代码仓库

**需要的三个核心变量**：

HOST：服务器 IP（例如 47.79.xx.xx）  
USERNAME：服务器登录用户（如 root / admin）  
SSH_KEY：服务器私钥（用于免密登录）

---

### 4. SSH 远程执行机制
作用：
- GitHub Actions 通过 SSH 登录服务器
- 然后执行 shell 命令

**依赖关系**：

服务器端：
~/.ssh/authorized_keys 存放公钥

GitHub端：
Secrets 存放私钥

---

### 5. 服务器运行环境（Node + PM2）

**必须具备**：

Node.js（通过 nvm 安装）  
npm（包管理器）  
pm2（守护进程工具）

---

### 6. PM2（进程守护工具）
作用：
- 保持 Node 服务一直运行
- 服务器重启后自动恢复
- 提供日志查看

常用操作：

pm2 start 项目
pm2 restart 项目
pm2 list 查看进程
pm2 logs 查看日志

---

## 三、deploy.yml 完整流程说明

**核心作用**：
当 GitHub main 分支更新时，自动部署到服务器

执行逻辑如下：
```js
name: deploy backend

on:                         //监视
  push:                     //push更新
    branches: ["main"]      //监视main分支

jobs:                       //定义任务
  deploy:                    
    runs-on: ubuntu-latest

    steps:
      - name: SSH deploy
        uses: appleboy/ssh-action@v1.0.3           //使用插件SSH登录服务器
 
        with:
          host: ${{ secrets.HOST }}                  
          username: ${{ secrets.USERNAME }}              
          key: ${{ secrets.SSH_KEY }}             //三个数据都存在github的Secrets(仓库密钥)中

          script: |                              //脚本
            # 1. 加载 Node 环境（nvm） 
            export NVM_DIR="$HOME/.nvm"
            source "$NVM_DIR/nvm.sh"

            # 2. 进入项目目录
            cd ~/interview-study-hub

            # 3. 拉取最新代码
            git pull origin main

            # 4. 安装依赖（更新 package）
            npm install

            # 5. 重启后端服务（让新代码生效）
            pm2 restart interview-backend
```
