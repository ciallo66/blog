# python的学习
>Python 注重开发效率，语法简洁，动态类型语言，不会提前检查类型，开发更方便快捷
>目前 Python 主要应用于人工智能、数据分析、自动化脚本、爬虫、Web 后端开发等领域，尤其在 AI 等智能体研发有独有的优势。

## 一.基础语法
### 1. 变量与数据类型
Python 不需要提前声明变量类型，直接赋值即可。常用数据类型主要有：
| 类型    | 作用   |
| ----- | ---- |
| int   | 整数   |
| float | 小数   |
| str   | 字符串  |
| bool  | 判断真假 |
| None  | 表示空值 |

**注意：**
* Python 是动态类型语言，变量类型会根据赋值自动变化。
* 不同类型之间经常需要转换，例如字符串转数字。

### 2. 条件判断与循环
Python 通过条件和循环控制程序执行流程。
| 功能               | 作用         |
| ---------------- | ---------- |
| if / elif / else | 根据条件执行不同代码 |
| for              | 遍历数据       |
| while            | 满足条件持续执行   |
| break            | 结束循环       |
| continue         | 跳过当前循环     |

**注意：**
* Python 使用缩进表示代码层级，不使用大括号。
* 循环主要用于处理列表、文件、数据等批量操作。

## 二.数据结构
Python 常用数据结构主要包括列表（list）、字典（dict）、元组（tuple）、集合（set），用于不同场景的数据存储和处理。(只有列表和元组可以进行下标访问)

### 1. 列表（list）
列表用于存储一组有序、可修改的数据，是开发中最常用的数据结构。

基本格式：
```python
list_name = [数据1, 数据2, 数据3]
```

常用操作：
```python
list.append()    # 添加数据
list.remove()    # 删除数据
list[index]      # 根据索引获取数据
list[index] = x  # 修改数据
```
常用于保存用户列表、商品列表等需要遍历和修改的数据。

### 2. 字典（dict）
字典通过 key-value（键值对）保存数据，适合根据标识快速获取信息。

基本格式：
```python
dict_name = {
    "key": "value"
}
```

常用操作：
```python
dict["key"]          # 获取数据
dict["key"] = value  # 修改或添加数据
dict.get("key")      # 安全获取数据
```
实际开发中大量使用，例如接口数据、用户信息、配置文件等。
### 3. 元组（tuple）
元组用于保存不可修改的数据。

基本格式：
```python
tuple_name = (数据1, 数据2)
```

特点：
* 有序
* 不可修改

适合保存固定数据。
### 4. 集合（set）
集合用于存储不重复的数据，常用于去重。

基本格式：
```python
set_name = {数据1, 数据2}
```

常用操作：
```python
set.add()       # 添加数据
set.remove()    # 删除数据
```
### 5. 实际开发选择
| 数据结构  | 主要用途          |
| ----- | ------------- |
| list  | 管理一组可变化的数据    |
| dict  | 根据 key 快速查询数据 |
| tuple | 保存固定不变的数据     |
| set   | 数据去重          |

实际开发中最常用的是 **list 和 dict**，尤其是在接口开发、数据处理、业务逻辑中大量出现。

## 四.面向对象编程（OOP）
Python 是一种支持面向对象编程的语言，OOP 主要用于将数据和功能封装在一起，让代码结构更加清晰，提高代码复用性和维护性。在实际项目中，常用于管理用户、商品、订单等具有属性和行为的业务对象。
### 1. 类与对象
类是创建对象的模板，对象是类创建出来的具体实例。
基本格式：
```python
class 类名:
    def 方法名(self):
        代码
```

主要概念：
| 内容         | 说明                  |
| ---------- | ------------------- |
| 类（class）   | 定义对象的结构和功能          |
| 对象（object） | 根据类创建的具体实例          |
| 属性         | 对象保存的数据，例如用户名称、商品价格 |
| 方法         | 对象可以执行的操作           |

### 2.鸭子类型
| 内容       | 说明                                                    |
| -------- | ----------------------------------------------------- |
| 定义       | 不关注对象是什么类型，只关注对象是否具有需要的方法                             |
| 核心思想     | “只要行为像，就可以当作同一种对象使用”                                  |
| 特点       | 不进行严格类型检查，提高代码灵活性                                     |
| 示例       | 函数需要 `speak()` 方法，不管对象是 Dog 还是 Cat，只要有 `speak()` 就能调用 |
| Python特点 | Python 是动态类型语言，天然支持鸭子类型                               |


## 五.文件与异常处理
### 1. 文件操作
基本格式：
```python
with open("test.txt") as f:
    data = f.read()
```
常用方式：
| 语法/方法 | 作用 | 示例 |
| --- | --- | --- |
| `with open()` | 自动打开和关闭文件（推荐） | `with open("test.txt") as f:` |
| `"r"` | 读取文件（默认） | `open("a.txt","r")` |
| `"w"` | 写入文件（覆盖原内容） | `open("a.txt","w")` |
| `"a"` | 追加写入 | `open("a.txt","a")` |

| `write()` | 写入内容 | `f.write("hello")` |

**注意**：
1. 读取和写入必须要经过open()打开文件
2. json的操作是自动序列化和反序列化

### 2. 异常处理
异常处理用于捕获程序运行中的错误，避免程序因为异常直接终止,和js里面的try...catch类似。  
基本格式：
```python
try:
    执行代码
except:
    错误处理
```

主要结构：
| 关键字     | 作用         |
| ------- | ---------- |
| try     | 执行可能出错的代码  |
| except  | 捕获并处理异常    |
| finally | 无论是否异常都会执行 |

## 六.迭代器
### 1.内置/容器迭代器
**特点**：
1. 一次性（只能向前，不能回头）
2. 省内存（按需取值）
#### (1)可迭代对象（Iterable）
可一被 `for ... in ... `循环遍历，或者能传给 iter() 函数的对象，都是可迭代对象
常见的有：list（列表），dict（字典），range()，str（字符串），open() 文件对象

#### (2)迭代器
1. iter(obj)：传入一个“可迭代对象”，返回对应的“迭代器”（相当于把书签放到书的第一页）。
2. next(it)：传入“迭代器”，向后翻一页并吐出数据。
```python
# 1. 这是一个普通的列表（可迭代对象）
fruits = ["苹果", "香蕉", "西瓜"]

# 2. 通过 iter() 拿到它的【迭代器】
fruits_iterator = iter(fruits)

# 3. 每次用 next() 拿出一个元素
print(next(fruits_iterator))  # 输出: 苹果
print(next(fruits_iterator))  # 输出: 香蕉
```



### 2.自定义类迭代器（原理级）
1. __iter__(self)：告诉 Python“我就是一个迭代器”，直接返回 return self。
2. __next__(self)：定义“怎么拿到下一个数”。如果拿完了，用 raise StopIteration 告诉系统停止。

```python
class MyCounter:
    def __init__(self, max_num):
        self.max_num = max_num
        self.current = 1

    # 1. 必须有：返回自己
    def __iter__(self):
        return self

    # 2. 必须有：定义每次怎么吐出下一个值
    def __next__(self):
        if self.current <= self.max_num:
            val = self.current
            self.current += 1  # 指针往后移一位
            return val
        else:
            # 数据用光了，抛出异常叫停循环
            raise StopIteration
# --- 测试使用 ---
counter = MyCounter(3)

# 像使用普通列表一样用 for 循环遍历它！
for num in counter:
    print(num)
```

### 3. 生成器迭代器(语法糖)
**介绍**：生成器是一个特殊的迭代器，是一个会自动保存执行状态的函数，而 yield 就是负责"返回数据 + 保存现场 + 下次继续执行"的关键字
1. return 返回一个值后立即结束函数，后面的代码不会再执行。
2. yield 返回一个值后不会结束函数，而是暂停执行并保存当前状态。
3. 下次调用 next() 时，生成器会从上次 yield 暂停的位置继续执行。

### 4.标准库高效迭代器（itertools 工具箱）

| 函数 | 作用 | 示例 | 输出示例 | 常见场景 |
|------|------|------|----------|----------|
| `count(start=0, step=1)` | 无限计数器 | `count(1)` | `1 2 3 4 ...` | 自增 ID、编号、无限循环 |
| `chain(*iterables)` | 拼接多个可迭代对象 | `chain([1,2],[3,4])` | `1 2 3 4` | 合并多个列表、多个查询结果 |
| `islice(iterable, stop)` | 对迭代器进行切片 | `islice(g, 5)` | `0 1 2 3 4` | 截取生成器、分页读取、限制遍历数量 |
| `combinations(iterable, r)` | 返回所有不重复组合 | `combinations([1,2,3],2)` | `(1,2) (1,3) (2,3)` | 排列组合、算法、面试题 |
| `product(*iterables)` | 返回笛卡尔积 | `product("AB",[1,2])` | `('A',1) ('A',2) ('B',1) ('B',2)` | 多层循环、参数组合、生成测试数据 |
| `cycle(iterable)` | 无限循环遍历序列 | `cycle(["A","B","C"])` | `A B C A B C ...` | 轮播、循环任务、重复数据 |
| `zip_longest(*iterables)` | 长度不同也能同时遍历 | `zip_longest([1,2],["A"])` | `(1,'A') (2,None)` | 合并长度不同的数据、数据对齐 |

## 七.装饰器
### 1.普通写法
**介绍**：装饰器用于在不修改原函数代码的前提下，为函数动态添加额外功能。
**本质**：装饰器本质上利用闭包和高阶函数，对函数进行包装，返回增强后的新函数，实现功能增强和代码复用。
```python
def decorator(func):      # 加工环境，接收原函数
    def wrapper():        # 加工厂
        print("开始")      
        func()            # 原函数
        print("结束")      
    return wrapper        # 返回加工后的函数
# wrapper 新函数替代了原先的函数 func

```
### 2.语法糖 @
**介绍**： @ 是装饰器的语法糖，会在函数定义完成后自动将函数传入装饰器进行包装（相当于自动执行 函数 = decorator(函数)），使代码更加清晰、易读、易维护。
```python
@decorator
def hello():
    print("Hello")
#不使用语法糖还需要 hello = decorator(hello)
```
