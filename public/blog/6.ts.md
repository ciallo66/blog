## 二.常用类型
### 1.any
**作用**：表示任何类型，使用时不需要类型检查  
**注意**：可赋值给任意类型的变量，可能会影响其他变量，少用

### 2.unknow
**作用**：安全的表示任何类型，使用前需要类型检查
#### (1)进行if判断
```ts
if(typeof a === 'string'){
    x = a
}
```
#### (2)断言
```ts
//第一种写法
x = a as string
//第二种写法
x = <string> a
```

### 3.never
**作用**：不能有值
1. 不同于限制变量
2. 一般由ts推断出来的
3. 限制函数返回值，该函数不能正常结束

### 4.void
**作用**：限制函数返回值，可以是显式和隐式undefined，结果不可被调用
1. 从语法上讲：函数是可以返回 undefined 的，至于显示返回，还是隐式返回，这无所谓！
2. 从语义上讲：函数调用者不应关心函数返回的值，也不应依赖返回值进行任何操作！即使返回了 undefined 值。

### 5.tuple
**作用**:本质是一个特殊的数组，可以存储固定数量的元素，类型已知且可以不同(不是一种关键词，是一种写法)  
有三种写法：
```js
// 1.第一个元素必须是 string 类型，第二个元素必须是 number 类型。
let arr1: [string, number]

// 2.第一个元素必须是 number 类型，第二个元素是可选的，如果存在，必须是 boolean 类型。
let arr2: [number, boolean?]

// 3.第一个元素必须是 number 类型，后面的元素可以是任意数量的 string 类型
let arr3: [number, ...string[]]
```

### 6.enum
**作用**:枚举(enum)可定义一组命名常量，增加代码可读性，让代码更好维护
#### (1)数字枚举
**作用**：其成员的值会自动递增，具备**反向映射**的特点(额外新增一对键值反过来数据)，其中的数据是**只读**的，不可修改！！
```js
TypeScript
enum Direction {
    Up = 0,       //可自定义值，默认从0开始递增
    Down,
}

// 反向映射
console.log(Direction.Up)
console.log(Direction.Direction[0])
```

#### (2)字符串枚举
作用：和数字枚举类似，但是没有反向映射
```js
enum Direction {
    Up = 'Up',       
    Down = 'Down',
}
```

#### (3)常量枚举
**作用**：是特殊枚举，用const定义，在编译时会**内联**，避免生产额外代码
**内联**：内联就是 TypeScript 在编译时，会将枚举成员引用替换为它们的实际值，而不是生成额外的枚举对象。这可以减少生成的 JavaScript 代码量，并提高运行时性能。
```ts
//如果不调用的话编译成ts是空白的，如果调用只显示调用的结果，不显示num对象
const num Direction {
    Up = 'Up',       
    Down = 'Down',
}
```

## 三.常用语法
### 1.type
**作用**：type 可以为任意类型创建别名，让代码更简洁、可读性更强，同时能更方便地进行类型复用和扩展。
#### (1)联合类型
```ts
//可以表示一个值是不同类型之一
type Mima = number | string 
type Gender = '男' | '女'
```

#### (2)交叉类型
```ts
//可以将多个类型合并成一个类型
type People= {age: number ,mame: string} 
type Address= {num:number ,room: number}
type House= People & Address
```

### 2.特殊情况
- 使用类型声明限制函数返回值为 void 时，TypeScript 并不会严格要求函数返回空
- 目的是为了不影响这类功能：箭头函数简写形式，写成一行，对应的代码就是返回值，导致void报错
```ts
type LogFunc = () => void

const f1: LogFunc = () => {
  return 100; // 允许返回非空值
};
```




### 3.接口(interface)
**作用**:interface 是一种定义结构的方式，主要作用是为：类、对象、函数等规定一种契约，确保代码的一致性和类型安全，注意 interface 只能定义格式，不能包含任何实现！
**使用场景**：
1. **定义对象的格式**：描述数据模型、API 响应格式、配置对象……等等，是开发中用的最多的场景。
2. **类的契约**：规定一个类需要实现哪些属性和方法。
3. **自动合并**：一般用于扩展第三方库的类型，这种特性在大型项目中可能会用到。


#### (1)三种应用场景
```ts
//定义一个接口
interface PersonInterface {
  name: string
  age: number
  speak(n: number): void
}

//1.定义类结构 (使用implements连接类和接口)
class Person implements PersonInterface{
    ······
}

//2.定义对象结构 (直接吧接口当做类型使用)
const user: PersonInterface = {
    ······
}

//3. 定义函数结构 (像使用type一样使用)
interface CountInterface {
  (a: number, b: number): number;
}
const count: CountInterface = (x, y, z) => {
  return x + y
}
```

#### (2)接口之间的继承
可以使用`extends`来继承别的接口，新的接口需要需要满足自己的和集成过来的
```ts
interface PersonInterface {
  name: string // 姓名
  age: number // 年龄
}

interface StudentInterface extends PersonInterface {
  grade: string // 年级
}
```

#### (3)接口的可合并性
重复定义同一个名字的接口，ts底层会进行合并，结果就是都要满足
```ts
interface PersonInterface {
  name: string // 姓名
  age: number // 年龄
}

interface PersonInterface {
  gender: string
}
```

### 4.泛型
**作用**：可以在定义函数、类或接口时，使用类型参数来表示未指定的类型，在具体使用是才进行定义。能让同一段代码适用于多种类型的情况下保证安全。
#### (1)泛型函数
```ts
function logData<T,U> (data1: T,data2: U): T | U{
  return Date.now() % 2 ? data1 : data2
}
logData<number,string>(100,'偶数')
logData<string,boolean>('你好',false)

```
#### (2)泛型接口
```ts
interface Personinterface<T> {
  name: string,
  extraInfo: T
}
let p: Personinterface<number> = {
  name:'张三',
  extraInfo:113
}
```

#### (3)泛型约束
可以使用`extends`来约束泛型
```ts
type aaa ={
  name:string
}
interface bbbinterface<T extends aaa> {
  name:T
}

```

## 四.相似内容区别
### 1.interface 与 type 的区别
**相同点**：interface 和 type 都可以用于定义对象结构，两者在许多场景中是可以互换的。  
**不同点**： 
- interface：更专注于定义对象和类的结构，支持继承、合并。
- type：可以定义类型别名、联合类型、交叉类型，但不支持继承和自动合并。

### 2.interface 与抽象类的区别
**相同点**：都用于定义一个类的格式（应该遵循的契约）。
**不同点**：
- 接口：只能描述结构，不能有任何实现代码，一个类可以实现多个接口。
- 抽象类：既可以包含抽象方法，也可以包含具体方法，一个类只能继承一个抽象类。
