## 2.fs模块
```js
//导入 fs 模块
const fs = require('fs')
```
### (1)写入文件
#### 1.一次性写入
**语法**：`fs.writeFile(file, data, [options], callback)`(异步写法)  
`writeFileSync`(同步写法)  

**参数说明**:
- file 文件名
- data 待写入数据
- options 选项配置 (`{flag:'a'}`,w:写入,a:追加,r:读取)
- callback 回调 


```js
fs.writeFile('./座右铭.txt','\r\n海纳百川，有容乃大。',err => {     // \r\n是换行
    if(err){
        console.log('写入失败');
        return;
    }
    console.log('写入成功');
});

```

#### 2.追加写入
**代码**:fs.appendFile(file, data, [options], callback)(异步写入)  
fs.appendFileSync(同步写入)  
>格式同上

#### 3.流式写入
**优点**:一段一段写入，适合写入大文件，占用内存小  
**代码**:fs.createWriteStream();
```js
const ws = fs.createWriteStream('./数据.txt');
ws.write('data1\r\n');
ws.write('data2\r\n');
ws.write('data3\r\n');

ws.close();       //关闭通道  
```

## 4.HTTP组成
**组成**:请求行 + 请求头 + 请求体

### (1)请求行
**组成**:请求方法 + URL + HTTP版本号

**URl**:协议名 + 主机名 + 端口号 + 路径 + 查询字符串

### (2)请求头
**组成**:各种键值对

### (3)请求体

## 6.HTTP模块

### (1)创建服务器
**代码**:`const server = http.createServer((request, response)=>{})`  
>request(请求):客户端发给服务器的内容，就是请求报文。  
>response(响应):服务器返回给客户端的内容，就是响应报文。  

```js
//创建一个最简单的服务器
//1. 导入 http 模块
const http = require('http');

//2. 创建服务对象
const server = http.createServer((request, response) => {
  response.end('Hello HTTP'); //设置响应体
});

//3. 监听端口, 启动服务
server.listen(9000, () => {
  console.log('服务已经启动....')
});
```

### (2)获取HTTP请求报文
想要获取请求的数据，需要通过 `request` 对象

| 含义           | 语法                                                     | 重点掌握 |
| -------------- | -------------------------------------------------------- | -------- |
| 请求方法       | `request.method`                                         | \*       |
| 请求版本       | `request.httpVersion`                                    |          |
| 请求路径       | `request.url`                                            | \*       |
| URL 路径       | `require('url').parse(request.url).pathname`             | \*       |
| URL 查询字符串 | `require('url').parse(request.url, true).query`          | \*       |
| 请求头         | `request.headers`                                        | \*       |
| 请求体         | `request.on('data',function(chunk){})`<br>`request.on('end',function(){});` |          |


***注意事项***:
1.  `request.url` 只能获取路径以及查询字符串，无法获取 URL 中的域名以及协议的内容
2.  `request.headers` 将请求信息转化成一个对象，并将属性名都转化成了『小写』
3.  关于路径：如果访问网站的时候，只填写了 IP 地址或者是域名信息，此时请求的路径为『`/`』
4.  关于 `favicon.ico`：这个请求是属于浏览器自动发送的请求

### (3)设置HTTP响应报文

| 作用               | 语法                                                         |
| ------------------ | ------------------------------------------------------------ |
| 设置响应状态码     | `response.statusCode`                                        |
| 设置响应状态描述   | `response.statusMessage`（用的非常少）                       |
| 设置响应头信息     | `response.setHeader('头名', '头值')`                         |
| 设置响应体         | `response.write('xx')` <br> `response.end('xxx')`            |

>write 和 end 的两种使用情况：

```javascript
//1. write 和 end 的结合使用 响应体相对分散
response.write('xx');
response.write('xx');
response.write('xx');
response.end(); //每一个请求，在处理的时候必须要执行 end 方法的

//2. 单独使用 end 方法 响应体相对集中
response.end('xxx');
```

### (4)设置资源类型（mime类型）

媒体类型（通常称为 Multipurpose Internet Mail Extensions 或 MIME 类型）是一种标准，用来表示文档、文件或字节流的性质和格式。

**mime** 类型结构： `[type]/[subType]`
>例： `text/html` `text/css` `image/jpeg` `image/png` `application/json`

HTTP 服务可以设置响应头 `Content-Type` 来表明响应体的 MIME 类型，浏览器会根据该类型决定如何处理资源

下面是常见文件对应的 mime 类型
```javascript
html: 'text/html',
css: 'text/css',
js: 'text/javascript',
png: 'image/png',
jpg: 'image/jpeg',
gif: 'image/gif',
mp4: 'video/mp4',
mp3: 'audio/mpeg',
json: 'application/json'
```


## 7.模块化
### (1)暴露数据

模块暴露数据的方式有两种：

1.  `module.exports = value`
2.  `exports.name = value`

<br>

使用时有几点注意：
1. `module.exports` 可以暴露 **任意** 数据
2. 不能使用 `exports = value` 的形式暴露数据，模块内部 `module` 与 `exports` 的隐式关系：
3. `exports = module.exports = {}`

### (2)导入模块

- 在模块中使用 require 传入文件路径引入文件
```js
const test = require('./me.js');
```

require 使用的一些注意事项:
1.  对于自己创建的模块，导入时路径建议写相对路径，且不能省略 ./ 和 ../
2.  js 和 json 文件导入时可以不用写后缀，同名不同后缀优先导入顺序为 js > json > node
3.  如果导入其他类型的文件，会以 js 文件进行处理
4.  如果导入的路径是个文件夹，则会首先检测该文件夹下 package.json 文件中 main 属性对应的文件，如果 main 或 package.json 属性不存在，则会检测文件夹下的 index.js 和 index.json，如果还是没找到，就会报错

## 8.包的管理工具的简单介绍
### (1)下载安装包

我们可以通过 `npm install` 和 `npm i` 命令安装包。

## 格式
```bash
npm install <包名>
npm i <包名>
```

运行之后文件夹下会增加两个资源：
- `node_modules` 文件夹：存放下载的包
- `package-lock.json` 包的锁文件，用来锁定包的版本

**关于依赖**：安装 `uniq` 之后，`uniq` 就是当前这个包的一个**依赖包**，有时会简称为**依赖**。
比如我们创建一个包名字为 A，A 中安装了包名字是 B，我们就说 **B 是 A 的一个依赖包**，也会说 **A 依赖 B**。
