# Ajax(fetch,Axios)

>什么是Ajax：不刷新整个页面，异步请求数据向服务器发送/接收数据。

## 一.fetch请求
### 1、概念
- 浏览器内置的基于Promise的 HTTP 请求 API，返回值始终是一个 Promise。

### 2、基本语法
    fetch(url, options?)

### 3、使用格式
``` js
fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },    //添加一个请求头，告诉后端前端请求的是一个Json字符串
    body: JSON.stringify({ name: '张三' })       // 如果传参，必须手动转成纯字符串
    })                                       //options选填
    .then(res => {                       //手动检查res.ok
        if (!res.ok) throw new Error(`HTTP ${res.status}`);//返回的response为false，终止函数，调用最近的.catch
        return res.json();                         //数据正常将response转换成json格式，数据传送到后面的.then调用
    })
    .then(data => console.log(data))
    .catch(err => console.error(error))
    .finally(() => { console.log('结束') });
```

<details>
<summary><h3>四、常用 options 参数</h3></summary>

| 字段 | 类型 | 说明 |
|------|------|------|
| method | string | 请求方法，默认 GET |
| headers | object | 请求头 |
| body | string/FormData/Blob | 请求体（GET 不可用） |
| credentials | string | 'include' 跨域携带 cookie |
| mode | string | 'cors' / 'no-cors' |
| cache | string | 缓存策略 |
</details>

<details>
<summary><h3>五、常用Response 对象的属性与方法</h3></summary>

| 名称 | 说明 |
|------|------|
| ok (boolean) | 状态码 200‑299 时为 true |
| status (number) | HTTP 状态码 |
| json() → Promise | 解析为 JSON |
| text() → Promise | 解析为字符串 |
</details>

### 六、总结
- **fetch** 只在网络错误时返回错误信息 reject调用.catch，HTTP 路径错误需手动检查 `res.ok` 并 throw。
- **res.ok** 是一个布尔值属性，表示 HTTP 响应状态码是否在 200-299 范围内。如果是，则为 true；否则为 false
- **throw** 是 js 的抛出异常语句，后面跟着一个错误对象（如 new Error(...)）。执行 throw 会立即中断当前函数执行，并将错误传递给最近的 catch 块。
>网络错误：fetch 的 Promise 会直接变为 rejected（拒绝），直接调用.catch  
>http状态码错误：fetch 识别不到， Promise 仍然是 resolved，.调用.then判断res.ok，从而调用.catch  

## Axios
### 1.初始化
```js
//1.安装
npm install axios

//2.导入
const axios = require('axios')
```

### 2.基础语法(crud)
**格式**：  
axios.post(url,[data], [config])
>get，delete没有data数据

| 方法 | 作用 | 典型场景 | 是否幂等 | 请求体（body） |
|------|------|----------|----------|----------------|
| **GET** | 获取资源 | 查询用户列表、获取文章详情 | ✅ 是 | ❌ 通常无 |
| **POST** | 创建资源 | 注册用户、发布新文章 | ❌ 否 | ✅ 包含创建的数据 |
| **PUT** | 完整替换资源 | 更新整个用户信息（全部字段覆盖） | ✅ 是 | ✅ 完整的资源数据 |
| **DELETE** | 删除资源 | 删除用户、删除文章 | ✅ 是 | ❌ 通常无 |
| **PATCH** | 部分更新资源 | 只修改用户姓名、只更改文章标题 | ✅ 通常设计为幂等 | ✅ 包含要修改的字段 |


### 3. 常用请求配置项表(config)
| 配置项 | 作用 |
|--------|------|
| `url` | 请求的接口地址（如 `'/api/users'`） |
| `method` | HTTP 方法（`'get'`、`'post'`、`'put'`、`'delete'` 等） |
| `baseURL` | 基础 URL，自动拼接在 `url` 前面（例如 `'http://api.example.com'`） |
| `headers` | 自定义请求头（例如 `{ Authorization: 'Bearer token' }`） |
| `params` | URL 查询参数（自动拼接成 `?key=value` 的形式） |
| `data` | 请求体内容（POST/PUT 时发送的数据，通常是一个对象） |
| `timeout` | 超时时间（单位毫秒，例如 `5000` 表示 5 秒没响应就中断） |
| `withCredentials` | 跨域请求时是否携带 Cookie（设为 `true`） |
| `responseType` | 期望服务器返回的数据类型（常用 `'json'`、`'blob'`、`'text'`） |

### 4.创建axios实例
**功能**:统一配置 baseURL、超时时间、请求头等，避免每个请求重复写。
```js
const api = axios.create({
  baseURL: 'http://api.example.com',
  headers: { 'X-Custom-Header': 'foobar' }
})

// 使用实例发送请求
api.get('/users')
```

### 5.拦截器

**功能**：在请求发送前或响应返回后统一执行逻辑，避免重复代码。常用于添加 token、统一错误处理、修改请求/响应数据。

#### (1)请求拦截器
在请求发送前执行，必须返回修改后的 `config`，否则请求不会发出。
```js
axios.interceptors.request.use(
    config => config,                          
    error => {
    return Promise.reject(error)
    }
)
```

#### (2)响应拦截器
在响应到达后执行，可以处理数据或统一处理错误（如 401 跳转登录页）。
```js
axios.interceptors.response.use(
    response => response.data,
    error => {
    if (error.response?.status === 401) {
        // 跳转登录页
    }
    return Promise.reject(error)
    }
)
```
#### 注意事项
- **必须返回值**：请求拦截器返回 `config`，响应拦截器返回 `response` 或 rejected Promise，否则链会中断。
- **执行顺序**：多个请求拦截器按添加顺序执行，响应拦截器按添加顺序**逆序**执行（后添加的先执行）。
- **异步支持**：拦截器内可使用 `async/await`，但最终必须返回正确类型的值。
- **全局与实例独立**：全局 `axios.interceptors` 不影响axios实例的请求和响应，每个实例也都可以写拦截器。
