# Vue 2 学习

## 一、基础语法
### 1.模板渲染类（指令与插值）
| 语法 | 全称/简写 | 作用 | 注意事项 |
|------|-----------|------|----------|
| 插值表达式 | `{{ }}` | 显示数据，支持表达式 | 会转义 HTML；不能用于属性 |
| v-text | 无简写 | 设置元素文本内容 | 会覆盖原有内容 |
| v-html | 无简写 | 输出原始 HTML | 小心 XSS(安全层面) |
| v-bind | `:` | 动态绑定属性，只向界面渲染单向 | 支持 class/style 增强语法 |
| v-show | 无 | 切换 display（显示/隐藏） | DOM 始终存在，适合频繁切换 |
| v-if | 无 | 条件渲染（创建/销毁） | 切换开销大，适合少切换 |
| v-else-if | 无 | 多分支条件 | 必须紧跟 v-if / v-else-if |
| v-else | 无 | 否则条件 | 必须紧跟 v-if / v-else-if |
| v-for | 无 | 循环渲染列表 | 一般必须绑定 `:key` |
| v-on | `@` | 绑定事件 | 支持修饰符，可传参 |
| v-model | 无 | 双向绑定，数据双向绑定 | 实质是 :value + @input |

### 2.数据逻辑与组件类

| 语法 | 写法 | 作用 | 注意事项 |
|------|------|------|----------|
| data | `data()` | 定义响应式数据 | 必须是函数 |
| computed | `computed: {}` | 计算属性（有缓存） | 数据变化才重新计算 |
| watch | `watch: {}` | 监听数据变化 | 适合异步或复杂操作 |
| props | `props: [...]` | 父传子数据，子组件使用接收数据 | 单向数据流 |
| $emit | `this.$emit()` | 子传父事件 | 事件名建议 kebab-case |
| slot | `<slot>` | 向组件传递模板 | 支持具名/作用域插槽 |
| components | components:{Nihao} | 声明组件(注册组件) | 局部优先 |
| 生命周期 | created/mounted 等 | 执行特定逻辑 | created 可操作数据，mounted 可操作 DOM |

### 3.其他常用语法与内置组件

| 语法 | 作用 | 注意事项 |
|------|------|----------|
| 事件修饰符 (.stop .prevent .once) | 防止冒泡，阻止默认事件，只执行一次 | 可链式使用 |
| 按键修饰符 (.enter .esc) | 指定按键触发 | 使用别名 |
| .lazy / .number / .trim | 修饰 v-model | 控制更新时机/类型 |
| `<keep-alive>` | 缓存动态组件 | 配合 `<component :is>` |
| `<transition>` | 动画过渡 | 配合 CSS 或 JS 钩子 |
| `<transition-group>` | 列表动画 | 每个项需唯一 key |
| 动态属性 `[attr]` | 绑动态属性名 | Vue 2.6+ 支持 |
| 动态组件 `<component :is>` | 按需渲染组件 | 可配合 keep-alive |

### 4.生命周期
- 定义：Vue在关键时刻帮我们调用的特殊名称函数(也叫生命周期钩子)
初始化数据前后
>beforeCreate
>created
编译模版放入DOM前后
>beforeMount
>mounted
数据更新前后
>beforeUpdate
>Updated
销毁前后，常用于解绑事件
>beforeDestroy
>destroyed

### 5.路由
- push和replace  
<RouterLink replace>加入replace可使当前界面跳转直接替换缓存，返回不到上一个界面  
例:bc路由跳转是repalce,a界面到b界面,到c界面,到d界面,返回上一界面直接到a

#### (1)路由的常规写法
- 需要新建一个路由配置项,基本写法如下
``` js
import Vue from 'vue'
import VueRouter from 'vue-router' 
import aaa from 'aaa'
import bbb from 'bbb'
Vue.use(VueRouter)

const router = new VueRouter({
    mode: 'hash',
    routes:[
        {
            name:'aaa',
            path:'/aaa',         //当跳转到当前界面
            component:aaa,      //调用aaa组件
            children:[
                {path:'/bbb',component:bbb}
            ]
        }
    ]
    })
```


- 同时需要在规定调用组件的所放的位置
- ##### 第一种写法，to使用模板字符串来写，传值用${}来写
```js
<router-link :to="`/blog/${post.id}`">{{ post.title }}</router-link>
```


- ##### 第二种写法，to使用对象写法，并传递数据
*使用query参数传递*
``` js
//发送方：
<router-link :to="{
    path:`/blog/${post.id}`,    //动态路径要使用模版字符串写法
    query:{ id:aaa.id }
    
    }">文章</router-link>
//接收方：
this.$route.query.id
```

*使用params传递参数(使用那么写法必须使用name，不能用path)*
``` js
<router-link :to="{
    //path:'/blog/aaa'(不能用)
    name:'aaa',               
    params: { id: m.id } 
    }">文章</router-link>

//路由配置
name:'aaa'
path:/blog/aaa/id   //这里的id就是传过来的值
//接收方
this.$route.params.id
```
        
        

#### (2)编程式路由(重要)
- `<router-link>`本质是转换成`<a>`标签，不需要`<a>`标签跳转就需要使用编程式路由  
- push：以堆栈的方式保存，可以返回上一个界面  
  replace：替换当前的路径缓存，无法返回上一个界面

```
tiaozhuan(){
    this.$router.push(`/blog/:id`)
}
//调用该函数就可以进行跳转，不用写<router-link>
```
