# 一些小知识点

### 1.RESTful API
RESTful API 是一种特殊风格的接口，主要特点有如下几个：
- URL 中的路径表示资源，路径中不能有动词，例如 `create`，`delete`，`update` 等这些都不能有
- 操作资源要与 HTTP 请求方法对应
- 操作结果要与 HTTP 响应状态码对应

规则示例：

| 操作 | 请求类型 | URL | 返回 |
| ---- | ---- | ---- | ---- |
| 新增歌曲 | POST | /song | 返回新生成的歌曲信息 |
| 删除歌曲 | DELETE | /song/10 | 返回一个空文档 |
| 修改歌曲 | PUT | /song/10 | 返回更新后的歌曲信息 |
| 修改歌曲 | PATCH | /song/10 | 返回更新后的歌曲信息 |
| 获取所有歌曲 | GET | /song | 返回歌曲列表数组 |
| 获取单个歌曲 | GET | /song/10 | 返回单个歌曲信息 |

### 2.res.json()作用
```js
// Express 底层简化的伪代码（它帮你写的）
res.json = function (obj) {
  let val = obj;

  // 1. 核心关键点：在后台悄悄咪咪调用 JSON.stringify()，把你的对象转成纯文本字符串！
  let body = JSON.stringify(val);

  // 2. 告诉浏览器：“哥们，我给你发的是标准的 JSON 文本，你准备好接货！”
  // （这就免去了你手动写头信息的痛苦）
  this.set('Content-Type', 'application/json');

  // 3. 啪的一声，把这串纯文本字符串通过网络真正发送给前端
  return this.send(body);
};
```
