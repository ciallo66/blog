# 一些小知识点

### 1.RESTful API
RESTful API 是一种特殊风格的接口，主要特点有如下几个：
- URL 中的路径表示资源，路径中不能有动词，例如 `create`，`delete`，`update` 等这些都不能有
- 操作资源要与 HTTP 请求方法对应
- 操作结果要与 HTTP 响应状态码对应

规则示例：

| 操作 | 请求类型 | URL | 返回 |
| ---- | ---- | ---- | ---- |
| 新增歌曲 | POST | /song | 返回新生成的歌曲信息 |
| 删除歌曲 | DELETE | /song/10 | 返回一个空文档 |
| 修改歌曲 | PUT | /song/10 | 返回更新后的歌曲信息 |
| 修改歌曲 | PATCH | /song/10 | 返回更新后的歌曲信息 |
| 获取所有歌曲 | GET | /song | 返回歌曲列表数组 |
| 获取单个歌曲 | GET | /song/10 | 返回单个歌曲信息 |

### 2.res.json()作用
```js
// Express 底层简化的伪代码（它帮你写的）
res.json = function (obj) {
  let val = obj;

  // 1. 核心关键点：在后台悄悄咪咪调用 JSON.stringify()，把你的对象转成纯文本字符串！
  let body = JSON.stringify(val);

  // 2. 告诉浏览器：“哥们，我给你发的是标准的 JSON 文本，你准备好接货！”
  // （这就免去了你手动写头信息的痛苦）
  this.set('Content-Type', 'application/json');

  // 3. 啪的一声，把这串纯文本字符串通过网络真正发送给前端
  return this.send(body);
};
```

### 5.Swap(虚拟内存)
**介绍**：当你的物理内存（RAM）不够用时，系统就会把内存里暂时不用的数据挪到硬盘swap  
**优点**：当内存挤爆时，它可以当防爆盾，防止系统直接崩溃或强杀你的进程（比如 MySQL）。  
**缺点**：硬盘的读写速度比物理内存慢几百倍。一旦系统频繁调用 Swap 里的数据，服务器就会明显变卡。  

#### 初始化
```js
//1.创建一个2GB 的 Swap
sudo dd if=/dev/zero of=/swapfile bs=1M count=2048 && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile

//2.让 Swap 分区永久生效
echo "/swapfile swap swap defaults 0 0" | sudo tee -a /etc/fstab && 

//3.永久修改 Swap 的使用比例（改为 30），并让它立刻生效(比例越大swap占用越大)
echo "vm.swappiness=30" | sudo tee -a /etc/sysctl.conf && sudo sysctl -p
```
