# uniapp的使用

## 一.uniapp的生命周期
UniApp 的生命周期可以分为三大类：**应用生命周期**、**页面生命周期**、**组件生命周期**。

### 1.应用生命周期（App Lifecycle）
应用生命周期作用于整个应用的运行过程，一般写在 App.vue 中，用来监听应用的启动、切换前后台以及异常情况。

1. onLaunch：应用首次启动时触发，只会执行一次，通常用于初始化全局数据、登录态校验或接口预加载。

2. onShow：应用从后台进入前台时触发，每次切回应用都会执行，常用于刷新数据或恢复状态。

3. onHide：应用从前台进入后台时触发，比如用户切换到其他应用或锁屏时触发，常用于保存数据或停止定时任务。

4. onError：应用发生运行错误时触发，用于捕获异常并进行日志记录或错误上报。


### 2.页面生命周期（Page Lifecycle）
页面生命周期作用于单个页面，用于控制页面的加载、显示、隐藏以及销毁过程，一般写在 pages 目录中的页面文件里。

1. onLoad：页面加载时触发，只执行一次，可以接收路由传递的参数，适合初始化页面数据。

2. onShow：页面每次显示时都会触发，例如从其他页面返回时也会执行，常用于刷新数据。

3. onReady：页面初次渲染完成后触发，此时页面结构已经生成，可以进行 DOM 或组件相关操作。

4. onHide：页面被隐藏时触发，例如跳转到其他页面时执行。

5. onUnload：页面被销毁时触发，例如返回上一级页面或关闭页面时执行。

### 3.组件生命周期（Component Lifecycle）
组件生命周期作用于单个组件，用来管理组件的创建、更新和销毁过程，适用于 components 目录中的自定义组件(以vue3为例)。
1. setup：初始化入口（替代created/beforeCreate）  

2. onBeforeMount：挂载前（DOM不可用）  

3. onMounted：挂载完成（DOM可用，常用请求/初始化） 

4. onBeforeUpdate：更新前（数据变了，视图未变）  

5. onUpdated：更新后（视图已刷新）  

6. onBeforeUnmount：销毁前（清理准备）  

7. onUnmounted：销毁完成（清理定时器/监听）

### 4.生命周期的执行顺序

1. App：控制应用是否运行（启动/前后台/错误）  
2. Page：控制当前在哪一页（加载/显示/离开）  
3. Component：控制页面里的模块（UI与局部状态）


## 二.uniapp和vue3语法差异对照表
### 1.布局结构类
| Vue3（Web） | UniApp | 说明 |
|------------|--------|------|
| div | view | 基础容器 |
| span | text | 文本节点 |
| img | image | 图片展示 |
| a | navigator | 页面跳转（非原生a标签） |
| ul / li | view + view | 无语义标签，纯结构替代 |

### 2.操作类
| event.target | event.detail | UniApp事件结构不同 |
|---|---|---|
| px | rpx | 响应式单位（小程序核心） |
| axios | uni.request | 内置请求方法 |
| fetch | uni.request | 替代方案 |
| localStorage | uni.setStorage / uni.getStorage | 跨端存储封装 |
| sessionStorage | uni.setStorageSync | 同步存储替代 |

### 3.路由与跳转类
| Vue3（Web） | UniApp | 说明 |
|------------|--------|------|
| vue-router.push | uni.navigateTo | 页面跳转 |
| router.replace | uni.redirectTo | 替换页面 |
| router.back | uni.navigateBack | 返回上一页 |
| router-link | navigator | 页面链接组件 |
