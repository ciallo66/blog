# SQL语法基础

## 一.sql分类
### 1. 介绍
1. DDL	数据定义语言，用来定义数据库对象 (数据库，表，字段)
2. DML	数据操作语言，用来对数据库表中的数据进行增删改
3. DQL	数据查询语言，用来查询数据库中表的记录
4. DCL	数据控制语言，用来创建数据库用户、控制数据库的访问权限

### 2. DDL (Data Definition Language) - 数据定义语言
* **核心作用：** 用于定义、修改或删除数据库的**物理结构和对象**（如库、表、视图、索引）。

| 操作类型 | 具体操作说明 | 官方标准 SQL 语法格式 |
| :--- | :--- | :--- |
| **创建** | 创建新的数据库或数据表 | `CREATE TABLE 表名 (列名 数据类型 约束, ...);` |
| **修改** | 修改已有表的结构（如添加/删除列） | `ALTER TABLE 表名 ADD/DROP COLUMN 列名 数据类型;` |
| **删除** | 彻底删除数据库或数据表（物理销毁） | `DROP TABLE 表名;` |
| **清空** | 清空表内所有数据，释放空间，但保留表结构 | `TRUNCATE TABLE 表名;` |


### 3. DML (Data Manipulation Language) - 数据操作语言
* **核心作用：** 用于对表中的**数据内容**进行动态管理（增、删、改），不影响表结构本身。

| 操作类型 | 具体操作说明 | 官方标准 SQL 语法格式 |
| :--- | :--- | :--- |
| **插入** | 往表中添加一行或多行新记录 | `INSERT INTO 表名 (列1, 列2) VALUES (值1, 值2);` |
| **更新** | 修改表中满足条件的已有数据 | `UPDATE 表名 SET 列1 = 值1 WHERE 条件;` |
| **删除** | 删除表中满足条件的某一行或多行数据 | `DELETE FROM 表名 WHERE 条件;` |


### 4. DQL (Data Query Language) - 数据查询语言
* **核心作用：** 专门用于从数据库中**检索、筛选、分组和排序数据**。

| 操作类型 | 具体操作说明 | 官方标准 SQL 语法格式 |
| :--- | :--- | :--- |
| **基础查询** | 从指定表中查询特定列的数据 | `SELECT 列1, 列2 FROM 表名;` |
| **行级筛选** | **分组前**，过滤出符合特定条件的原始数据 | `SELECT 列 FROM 表名 WHERE 条件;` |
| **数据分组** | 将数据按某一列或多列进行归类聚合 | `SELECT 列, COUNT(*) FROM 表名 GROUP BY 列;` |
| **组级筛选** | **分组后**，对聚合函数计算出的结果进行条件过滤 | `SELECT 列 FROM 表名 GROUP BY 列 HAVING 聚合条件;` |
| **结果排序** | 将最终查询结果按某列进行升序(ASC)或降序(DESC)排列 | `SELECT 列 FROM 表名 ORDER BY 列 DESC;` |
| **结果限制** | **所有处理完成后**，强行截断并指定最终只在屏幕上显示前 N 条数据 | `SELECT 列 FROM 表名 LIMIT 数量;` |
| **分页查询** | 跳过指定数量的行（偏移量），再连续往后读取指定数量的数据 | `SELECT 列 FROM 表名 LIMIT 跳过行数, 要看的行数;` |
 
**条件写法**： 
|写法|说明|
|---|---|
|age IN (20,21,22)        |查询年龄符合20或21或22的人                                |
|age BETWEEN 20 and 40    |查询年龄符合20到40区间的人，20和40不可互换，且包含20和40    |
|name LIKE '___'          |`_`代表一个字符，查询有姓名长度有三个字符的人             |
|name LIKE '张%'          |`%`代表任意数量的字符，查询姓张的人                    |

**书写顺序**：
SELECT -> FROM -> JOIN -> WHERE -> GROUP BY -> HAVING -> ORDER BY -> LIMIT

**执行顺序**：
FROM -> JOIN -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY -> LIMIT



## 二.功能表
### 1.功能表
| 概念 | 核心作用 |
|------|----------|
| **列** | 数据的基本单元，定义字段类型、默认值及是否可空 |
| **约束** | 保证数据的合法性和完整性（NOT NULL、UNIQUE、CHECK等） |
| **外键** | 维护跨表引用完整性，支持级联操作 |
| **引用** | 表间关系，连接父表与子表，形成实体依赖 |
| **触发器** | 在数据变更时自动执行逻辑（审计、校验、级联更新） |
| **索引** | 加速查询、排序和分组，实现唯一性约束 |
| **分区** | 水平拆分大表，优化管理、查询和维护效率 |
| **Statistics** | 查询优化器依此选择最佳执行计划 |
| **DDL** | 定义和修改数据库结构的语言（CREATE/ALTER/DROP） |
| **虚拟** | 基于表达式或查询的逻辑结构（虚拟列、视图），减少冗余 |

### 2.约束
|约束类型|是否允许为空|	是否允许重复	|表中数量	|核心用途|
|-|-|-|-|-|
|PRIMARY |KEY	 不允许	|不允许|	1 个	|唯一标识行|
|UNIQUE |KEY	 允许（多个 NULL 通常视为不同）|	 不允许|	多个	|保证列值唯一|
|CHECK	|看字段定义	|看条件	|多个	|限制数据范围 / 逻辑|
