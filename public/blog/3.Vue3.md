# Vue3新内容学习

## 1.ref vs reactive
| 类型 | 支持数据 | 读写方式 | 重新赋值 | 解构/传参 |
|------|----------|----------|----------|------------|
| ref | 基本数据,对象类型 | `.value` | 安全 | 安全 |
| reactive | 对象类型（数组、对象、Map、Set 等） | 直接 `.属性` |  丢失响应 |  丢失响应 |

- **注意** 
1. 在reactive里面的ref数据，调用输出的时候不需要.value ，reactive会自动帮ref解包
2. reactive定义的数据不可以整体替换，如由reactive定义的的obj = {???} ,或使用Object.assign(obj,{新对象})修改
3. ref定义基本数据基于Object.defineProperty，定义对象类数据基于proxy，也就是定义对象类在外面一层套了reactive，访问是.value.count
4. ref基本类型定义是基于Object.defineProperty，内部对原始值通过getters和setter拦截
**示例**：
```js
const count = ref(0)
count.value++

const obj = reactive({ a:1 })
obj.a++
// obj = { a:2 } 错误
```

## 2.computer

- **缓存**：依赖不变，多次访问不重新计算。
- **懒执行**：依赖变化且被读取时才重算。
- **返回值**：只读的 ref 对象。
- **可写**：用 `{ get, set }` 形式。

```js
// 只读
const double = computed(() => count.value * 2)

// 可写
const full = computed({
  get: () => first.value + last.value,
  set: (v) => { /* 拆分赋值 */ }
})
```

## 3.监视(watch)
### (1)watch
- 只能监视一下数据
>1.ref 定义的数据  
>2.reactive 定义的数据  
>3.函数返回的一个值(getter函数)  
>4.一个包含上述内容的数组  

- 特点：
**懒执行**：默认只在侦听源变化时执行，不会立即运行。  
**侦听源**：可以是 ref、reactive 响应式数据、getter 函数，或多个源的数组。  
**深度监听**：监听对象/数组内部变化需开启 `{ deep: true }`,用reactive默认深度监视,且无法关闭。  
**立即执行**：配置 `{ immediate: true }` 让回调初始时立即运行一次。  
**停止监听**：watch 函数返回一个停止函数，调用即可停止。  

```js
// 单个普通变量
watch(()=>count, (newVal, oldVal) => {})

// 对象里的某一个属性
watch(() => state.name, (newVal, oldVal) => {})

// 对数组里的属性进行监视，需要转换成函数，在需要高性能和需要旧值的时候用，
watch([()=>count, ()=>state.name], ([newCount, newName], [oldCount, oldName]) => {})

// 深度 + 立即
watch(()=>count, (newVal, oldVal) => {}, { deep: true, immediate: true })
```
### (2)watchEffect
- 特点：可以自动对操作的属性进行监视，不用写监视的属性，自带初始化，精准依赖追踪(精准深度追踪，不占用其他资源)
``` js
watchEffect(()=>{
    if(a>=6 || b>=4){
        console.log('BIG!')
    }
})
//自动监视a和b
```


## 5.hooks封装
- 1.创建文件/hooks/useSum.ts (一般以  use开头+作用  小驼峰写法)
- 2.导入所需的数据方法等并暴露
``` ts
import {ref,reactive} from 'vue'

export default function(){
    let sum = ref(0)
    function add(){
        sum.value +=1
    }
    return {sum,add}
}
```
- 3.最后在用到该封装内容的地方导入，然后分解构值
``` ts
<script setup name="Man" >
import useSum from '@/hooks/useSum';
const {sum,add} = useSum()
</script>
```

## 6.toRefs和toRef
- **作用**：如果普通解构赋值就会导致解构出的数据失去响应式，外面套一个toRefs或toRef就行了

- toRef:解构一个内容
- **特点**：可以解构对象里面没有的属性，不会报错，等有了的时候可以直接用
``` js
const user = reactive({ name: '张三', age: 18 })     

const nameRef = toRef(user, 'name')     //把user里的name解构出来

nameRef.value = '李四'                  //解构后仍然是ref，需要带.value
```

- toRefs:可以吧对象里的全部解构
- **特点**：可以一次性批量处理，解构的格式很直观
``` js
const user = reactive({ name: '张三', age: 18 })

const { name, age } = toRefs(user)

name.value = '王五'
```

## 7.vue3路由新写法
### (1)路由配置
- 工作模式的需要导入，创建路由也需要导入
```ts
import { createRouter ,createWebHashHistory,createMemoryHistory} from "vue-router";
import Home from '@/components/Home.vue'

const router = createRouter({
    history:createMemoryHistory(),        //工作模式
    routes:[
        {
            path:'/home',
            component:Home
        },  
    ]
})
export default router

//在main.ts里
import router from './router'
const app = createApp(App)      
app.use(router)
app.mount('#app')
```

### (2)路由的props
#### 将路由收到的所有**params**参数作为props传给路由组件(只能用params参数)
``` ts
name:'home'
path:'/home/:id/:title',
component:Home
props:true

//Home组件接收,收到的数据可以直接使用
defineProps(['id','title'])            
```
#### 函数写法，可以自己决定将什么作为props给路由组件
``` ts 
name:'home'
path:'/home/:id/:title',
props(route){                   //括号的里的参数就是route，直接返回query里面的数据
    return route.query              
}
//Home组件接收
defineProps(['id','title']) 
```
#### 对象写法，直接传递对象里的数据(数据是死的，一般不用)
```
props:{
    a:'100',b:'200'
}
```

import {useRouter} from 'vue-router'

## 8.组件通信
### (1)props (父传子)
- **作用**:父传子，子传父（最常用之一）

- 父传子
```ts
//父通过在子标签传递传递数据
<studnet :data="data" />

//子通过props声明接受数据
defineProps(['data'])
```
- 子传父
>本质是父传子一个函数，子函数调用，并把数据传到父那里
```ts
//本质是传递一个函数
<student :senddata="getData"/>
getData(value){
    console.log('我接受到了：',value);
}

//调用该函数传递数据
<button @click="sendData(valus)">点我向父传数据<button>  
defineProps(['sendData'])
```

### (2)自定义事件(子传父)
- **作用**：用与子传父
```ts
//父组件自定义事件aaa
<student @aaa="getData" />
getData(value){
    console.log('获得了数据:'，value)
}

//子组件声明事件
const emit = defineEmits(['aaa'])
<button @click="emit('aaa',1145)">点我向父传数据<button>    //emit(事件名,数据)
```

### (3)provide/inject(逐层传递)
```js
//爷组件
const msg = ref('爷数据')
// 核心：把 msg 变量以 'money' 的暗号传出去
provide('money', msg)

//孙组件
const myAsset = inject('money')
```

### (5)插槽（Slots）

* **核心价值**：将组件的**逻辑骨架**与**具体视觉呈现**彻底解耦。子组件负责提供公共样式和包装（挖坑），父组件决定往里面填什么内容（填坑）。
组件库（如 Element Plus）复用性的灵魂支撑。

#### 1.默认插槽（Default Slot）
```html
//插槽组件Card.vue
<div class="card">
    <h3>卡片标题</h3>
    <slot></slot> 
</div>

//父组件使用,直接把下面的内容放在<slot>标签里面
<template>        
    <Card>
        <button>一个按钮</button>
        <h3>一段话</h3>
    </Card>
</template>
```

#### 2.具名插槽
**作用**：东西多的时候，需要每一块都有自己的name管理,有一个默认名字 'default'
```html
//插槽组件Card.vue
<div class="card">
    <h3>卡片标题</h3>
    <slot name='s1'></slot> 
    <slot name='s2'></slot> 
</div>

//父组件使用
<Card>
    <template v-slot:s1>
        <h3>一堆内容</h3>
    </template>   
    <template #s2>
        <h3>总结内容</h3> 
    </template>   
</Card>
```
**注意**：具名插槽在父组件使用的时候，外面要套一层`<template>`标签，在里面写v-slot:?  
v-slot:s1可以简写成#s1


#### 3.作用域插槽
**作用**:数据在子组件，但结构（HTML 格式）由父组件决定。用于解决‘子组件拥有数据源、但父组件需要定制差异化渲染呈现’的场景。
```html
//插槽组件Card传递数据
<div class="card">
    <h3>卡片标题</h3>
    <slot :game="game" x="nihao1"></slot> 
</div>

//父组件接受
<Card>
    <template v-slot="params">         //或者直接解构赋值 v-slot="{game，x}"
        <h3 v-for="g in params.game" :key="g.id">{{g.game}}</h3>
    </template>
</Card>
```
**注意**：v-slot="params" 里的 params 是一个『数据全家桶』。
不管子组件在插槽上反向绑定了多少个属性，都会被 Vue 自动打包成一个统一的 JavaScript 对象塞给它。


## 9.其他API

### (1)shallowRef()和shallowReactive()
- **作用**：通过使用 shallowRef() 和 shallowReactive() 来绕开深度响应。
浅层式 API 创建的状态只在其顶层是响应式的，对所有深层的对象不会做任何处理，
避免了对每一个内部属性做响应式所带来的性能成本，这使得属性的访问变得更快，
可提升性能。

- 如何使用：直接包裹数据


### (2) toRaw和markRaw

* **`toRaw`**：将一个由 `reactive`等定义的响应式对象，还原为普通的原始 JavaScript 对象（脱掉 Proxy 外壳）。
* **`markRaw`**：标记一个普通对象，使其**永远无法**被转化为响应式对象。
- 如何使用：直接包裹数据

#### 1. toRaw
* **核心作用**：跳过 Proxy 的属性拦截，直接操作原始数据。
* **应用场景**：
  1. **性能优化**：在需要进行大量、高频的临时数据读取或写入，且**不希望触发任何视图更新（网页渲染）**时使用。
  2. **数据清理**：在将带有 Proxy 壳的数据作为参数发送给第三方非 Vue 库，或直接通过 API 发送给后端时，用于获取干净的纯净对象。

#### 2. markRaw
* **核心作用**：保护数据，防止不必要的数据变成响应式
* **应用场景**：
  1. **避免代理庞大实例**：如 Three.js、ECharts 或地图对象。这些实例内部结构极庞大，
  一旦被 Vue 递归进行 Proxy 拦截，会导致网页严重卡顿或内存溢出。
  2. **只读大数据**：成千上万条仅用于页面死字渲染、不需要任何动态点击交互的纯展示用数据。


### (3)customRef(自定义ref)

* **`customRef`**：**「自定义响应式」**。用于显式地控制一个变量的**依赖追踪（track）**和**触发更新（trigger）**时机。  
    **`customRef`** 要求必须返回一个带有 `get` 和 `set` 的对象
    **`customRef`** 通常用hooks进行封装处理

* **`track()`（收集依赖）**：
  * **触发时机**：当页面或代码**读取**（`get`）这个自定义 ref 的值时。
  * **核心作用**：告诉 Vue 记住当前是谁在用这个数据，对这个数据持续关注，一旦变化就更新
* **`trigger()`（触发更新）**：
  * **触发时机**：当代码**修改**（`set`）这个自定义 ref 的值时。
  * **核心作用**：当底层数据真正发生改变后，通知 Vue **立刻重新渲染**所有之前调用过 `track()` 的组件。


>案例：输入数据延迟1s后更新界面
```ts 
let initValue = '你好'          //一个初始值
let timer: number
let msg = customRef((track, trigger) => {
  return {
    //msg被读取时调用get
    get() {
      track()                   //track(跟踪)，告诉Vue要对msg进行持续关注，一旦msg变化就去更新
      return initValue
    },
    //msg被修改时调用set
    set(value) {
      clearTimeout(timer)       //关闭计时器，防抖
      timer = setTimeout(() => {
        initValue = value
        trigger()               //trigger(触发)，通知Vue一下数据msg变化了
      }, 1000);
    }
  }
})
```

### (4)Teleport

- **作用**:写登录弹窗时，为了不被父组件的样式和层级（z-index）给限制住，用 Teleport 直接把弹窗的本体发到最外层的 <body> 里，这样弹窗就能直接以整个屏幕为标准来做全屏居中显示了
```ts
//此时该组件的内容移到body里，以整个屏幕为标准做居中
<teleport to="body">
    <h3>请先登录</h3>
    <input type="text" placeholder="输入UID..." />
</teleport>
```
