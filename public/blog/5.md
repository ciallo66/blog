# 异步

## promise基础
### 一、概念
- Promise 是异步编程的解决方案，有三种状态：pending（等待）、fulfilled（成功）、rejected（失败）。
- 状态一旦改变（pending → fulfilled / pending → rejected），就不会再变。

### 二、创建 Promise
- 通过 `new Promise(executor)` 创建，executor 立即执行。
- executor 接收两个函数参数：`resolve`（将状态改为成功）和 `reject`（将状态改为失败）。
``` js
const myPromise = new Promise((resolve, reject) => {
    setTimeout(() => {
        const success = true; // 假设操作成功
        if (success) {
            resolve('操作成功！数据已获取');  // 3. 调用 resolve 表示成功
        } else {
            reject('操作失败');             // 4. 调用 reject 表示失败
        }
    }, 1000); // 延迟1秒
});
```

<details>
<summary><h3>三、实例方法（返回新 Promise，支持链式调用）</h3> </summary>

- `.then(onFulfilled, onRejected)`：成功时调用 onFulfilled，失败时调用 onRejected（通常只传一个成功回调，失败交给 .catch）。
- `.catch(onRejected)`：失败时调用，等价于 `.then(null, onRejected)`。
- `.finally(onFinally)`：无论成功失败都会执行，没有参数，不改变 Promise 状态。
``` js
myPromise
    .then(data => console.log(data))        //data由resolve('操作成功！数据已获取')传入
    .catch(error => console.error(error))     //errpr由reject('操作失败')传入
    .finally(() => console.log('请求结束（无论成功或失败）'));
```
</details>
<details>
<summary> <h3> 四、静态方法（用于组合多个 Promise）</h3>  </summary>

- `Promise.resolve(value)`：返回一个成功的 Promise。  
- `Promise.reject(reason)`：返回一个失败的 Promise。  
- `Promise.all(iterable)`：所有成功才成功，返回结果数组；任何一个失败则整体失败。  
- `Promise.race(iterable)`：返回第一个完成的 Promise（无论成功或失败）。  
- `Promise.allSettled(iterable)`：所有 Promise 完成后，返回每个结果的对象。  
- `Promise.any(iterable)`：返回第一个成功的 Promise；全部失败则返回 AggregateError。  
</details>

### 五、注意事项
- 链式调用中，任意一个 `.then()` 抛出错误，都会被后续最近的 `.catch()` 捕获。
- 一旦进入 `.then()` 或 `.catch()`，其后的代码会进入微任务队列，晚于当前同步代码执行。(也就是第一次宏任务之后执行微任务)
- 不要在 `resolve` 或 `reject` 之后写代码，它们不会终止 executor 的执行（但通常不再有意义）。

##  fetch请求

### 一、概念
- 浏览器内置的基于Promise的 HTTP 请求 API，返回值始终是一个 Promise。

### 二、基本语法
    fetch(url, options?)
### 三、使用格式
``` js
fetch(urlm,{
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
})                                       //options选填
    .then(res => {                       //手动检查res.ok
        if (!res.ok) throw new Error(`HTTP ${res.status}`);//返回的response为false，终止函数，调用最近的.catch
        return res.json();                         //数据正常将response转换成json格式，数据传送到后面的.then调用
    })
    .then(data => console.log(data))
    .catch(err => console.error(err))
    .finally(() => { console.log('结束') });
```

<details>
<summary><h3>四、常用 options 参数</h3></summary>

| 字段 | 类型 | 说明 |
|------|------|------|
| method | string | 请求方法，默认 GET |
| headers | object | 请求头 |
| body | string/FormData/Blob | 请求体（GET 不可用） |
| credentials | string | 'include' 跨域携带 cookie |
| mode | string | 'cors' / 'no-cors' |
| cache | string | 缓存策略 |
</details>

<details>
<summary><h3>五、常用Response 对象的属性与方法<h3></summary>

| 名称 | 说明 |
|------|------|
| ok (boolean) | 状态码 200‑299 时为 true |
| status (number) | HTTP 状态码 |
| json() → Promise | 解析为 JSON |
| text() → Promise | 解析为字符串 |
</details>

### 六、总结
- **fetch** 只在网络错误时返回错误信息 reject调用.catch，HTTP 路径错误需手动检查 `res.ok` 并 throw。
- **res.ok** 是一个布尔值属性，表示 HTTP 响应状态码是否在 200-299 范围内。如果是，则为 true；否则为 false
- **throw** 是 js 的抛出异常语句，后面跟着一个错误对象（如 new Error(...)）。执行 throw 会立即中断当前函数执行，并将错误传递给最近的 catch 块。
>网络错误：fetch 的 Promise 会直接变为 rejected（拒绝），直接调用.catch  
>http状态码错误：fetch 识别不到， Promise 仍然是 resolved，.调用.then判断res.ok，从而调用.catch  
.
