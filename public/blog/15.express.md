# express框架

## 一.express的初始化
1. 安装express
```js
npm i express
```
2. express基本写法
```js
//1.导入
const express = require('express');

//2.创建对象
const app = express();

//3.创建路由
app.get('/', (req, res) => {
    res.end('Ciallo');
});

//4.监听端口
app.listen(3000, () => {
    console.log('服务器已启动,http://localhost:3000');
});
```

## 二.express路由

### (1)路由的基本介绍
**路由的组成** :一个路由的组成有 `请求方法` ，`路径`，`回调函数`  
格式如下: 
>app.<method>(path ,callback)

### (2)路由方法
- **Express 框架**：为每个常用 HTTP 方法提供了便捷的路由方法（如 `app.get()`, `app.post()`），底层仍然基于 Node.js 的 `http` 模块。

下表列出最常用的 HTTP 方法及其在 Express 中的写法：

| HTTP 方法 | 用途（语义）          | Express 中的方法       | 是否幂等<br>(多次请求结果一致) | 请求体 (Body) | 
| --------- | --------------------- | ---------------------- |  ------------------------ | ---------- |
| **GET**   | 获取资源              | `app.get()`            |  ✅ 是                  | 无            |
| **POST**  | 创建新资源            | `app.post()`           | ❌ 否*                  | 有            | 
| **PUT**   | **完整**更新资源（替换） | `app.put()`            |  ✅ 是                | 有            | 
| **PATCH** | **部分**更新资源       | `app.patch()`          | ❌ 否*                 | 有            | 
| **DELETE**| 删除资源              | `app.delete()`         | ✅ 是                   | 可选          | 


## 三.请求与响应对象
### (1)获取请求报文

#### 1. 获取 URL 中的数据（GET / 通用）
| 核心代码 / 示例 | 类型 | 获取目标与核心语法 |
| :--- | :--- | :--- |
| `req.path;` | 属性 | **仅获取纯路径**：不带任何参数（例如 `req.url` 是 `/a?b=1`，则 `req.path` 仅为 `/a`）。 |
| `req.query;` | 属性 | **获取查询字符串（Query）**：全自动将 `?id=1&name=tom` 解析为 JS 对象 `{ id: '1', name: 'tom' }`。 |
| `req.params;` | 属性 | **获取路由动态参数（Params）**：配合动态路由 `/user/:id` 使用，可直接拿到 `{ id: 'xxx' }`。 |

#### 2. 获取请求体数据（POST / PUT）
| 核心代码 / 示例 | 类型 | 安装前提与核心作用 |
| :--- | :--- | :--- |
| `req.body;` | 属性 | **获取请求体（Body）**：常用于获取表单提交或 JSON 异步数据。**前提是必须在路由前配置对应的解析中间件**。 |

> **注意** **`req.body` 避坑中间件配置：**
> 如果直接读取 `req.body` 会得到 `undefined`！必须在你的 `app.js` 最上面加上这两行解析圣旨：
> ```javascript
> app.use(express.json()); // 解析客户端发来的 JSON 格式数据
> app.use(express.urlencoded({ extended: false })); // 解析客户端表单提交的 x-www-form-urlencoded 数据
> ```

#### 总结
* **`req.query` vs `req.params`**：
  * `query` 拿的是问号后面的参数（如 `/goods?id=10` ➔ `req.query.id`）。
  * `params` 拿的是路径的一部分（如路由设为 `/goods/:id`，路径为 `/goods/10` ➔ `req.params.id`）。
* **只读属性**：`req` 里面的属性绝大多数是只读的，用来接收前端传来的情报，不要手动去给它们赋值！


### (2)发送响应数据
#### 1. 基础响应与链式调用
| 核心代码 / 示例 | 类型 | 核心作用与避坑指南 |
| :--- | :--- | :--- |
| `res.status(500);` | 状态 | 设置 HTTP 状态码，支持链式调用。 |
| `res.set('xxx', 'yyy');` | 响应头 | 增强版设置。支持传入对象一次性设置多个响应头：`res.set({ 键: 值 })`。 |
| `res.send('中文不乱码');` | 响应体 | **【最常用】** 自动识别类型、**内置 UTF-8 防乱码**，且**自动结束响应**。 |
| `res.status(404).set('x','y').send('嗨')` | **链式** | **【大厂标配】** 一行代码优雅搞定“状态码 + 响应头 + 响应体”并发送。 |

#### 2. 特化业务响应
| 核心代码 / 示例 | 类型 | 核心作用与避坑指南 |
| :--- | :--- | :--- |
| `res.json({ name: 'tom' });` | API | **响应 JSON**：自动将对象/数组转为 JSON 字符串，并自动设响应头。 |
| `res.redirect('http://x.com');`| 跳转 | **重定向**：返回 302 状态码，强制浏览器自动跳转到新网址。 |
| `res.download('./file.json');` | 下载 | **触发下载**：将服务器本地文件作为“附件”吐给浏览器，触发全自动下载。 |
| `res.sendFile(__dirname + '/h.html');`| 渲染 | **响应文件**：直接读取本地 HTML、图片或媒体文件内容并渲染在网页上。 |

**注意**  
一次路由请求中，**结束响应的方法（如 `send`、`end`、`json`、`redirect`）只能成功执行一次**！
绝对不能在响应结束后再次尝试发送数据，否则后端服务器会当场抛出致命报错并挂掉：


## 四.express中间件(重要)

### (1)介绍
什么是中间件：
- 中间件（Middleware）本质是一个回调函数
- 中间件函数 可以像路由回调一样访问 请求对象（request）， 响应对象（response）


中间件的作用：
- 中间件的作用 就是 使用函数封装公共操作，简化代码

### (2)全局中间件
每一个请求 到达服务端之后 都会执行全局中间件函数
```js
let recordMiddleware = function(request, response, next){
    //实现功能代码
    //执行next函数(当如果希望执行完中间件函数之后，仍然继续执行路由中的回调函数，必须调用next)
    next();
}
app.use(recordMiddleware);
```
### (3)组件中间件
执行到对应的路由组件的时候，在路由组件运行前进行判断
```js
const myMiddleware = (req, res, next) => {
    console.log('--- 经过了中间件保安！ ---');
    next(); // 喊一声放行，直接去下一个地方！
};

app.get('/login', myMiddleware, (req, res) => {    //在组件里面加上这个中间件使用
    res.send('返回一个数据');
});
```

### (4)静态资源中间件
express 内置处理静态资源的中间件
```js
const path = require('path');
// 推荐使用 path.join 和 __dirname 锁死绝对路径，防止路径飘移 404
app.use(express.static(path.join(__dirname, 'public')));
```

**注意事项**：
1. index.html 文件为默认打开的资源
2. 如果静态资源与路由规则同时匹配，谁先匹配谁就响应
3. 路由响应动态资源，静态资源中间件响应静态资源
4. 谁先书写，优先匹配：如果静态资源与路由规则同时匹配，代码写在前面的先响应。
5. 中间件逻辑结束处必须确保能触发 next() 或 res.send() 终结，否则前端请求会永远处于转圈死锁状态。

## 五.防盗链
**作用**:通过中间件判断referer是不是在自己的网站，如果在别的网站就可以禁止别人调用自己的数据内容

```js
// 极简防盗链中间件演示
const antiStealMiddleware = (req, res, next) => {
    // 1. 抓出前端小车里的 Referer 标签（代表是谁在看我的模型）
    const referer = req.get('Referer');
    
    // 2. 如果标签里写的不是我自己的网站（比如不是 localhost:5173），直接拒绝！
    if (referer && !referer.includes('localhost')) {
        console.log(' 抓到小偷网站在盗用我的3D模型！！！');
        return res.status(403).send('拒绝访问！');
    }
    
    // 3. 检查通过，是我自己的网页在看，放行！
    next();
};
```


## 六.处理文件上传
### (1)核心概念与网络流
1. **前端载体 (`FormData`)**：前端唯一能混合打包二进制文件与文本数据的一个内置的容器。
2. **协议头 (`multipart/form-data`)**：
   - **注意**：前端发送 `FormData` 时，**绝不能手动设置 Headers**，必须由浏览器自动生成。

### (2)前端 HTML 属性限制
- **`accept="image/*"`**：货源过滤器。限制用户只能选择图片文件（可换成 `.gltf, .obj` 等后缀）。
- **`required`**：必选内容，未选文件时浏览器直接拦截提交。

### (3)后端 Express 核心组件
1. **`multer` (官方)**：
   - `const upload = multer({ dest: 'uploads/' })` ➔ 雇佣保镖，划定卸货仓库。
   - `upload.single('avatar')` ➔ 路由中间件闸门。只收单文件，前端 `name` 属性必须死死对应。
   - `req.file` ➔ 文件落盘成功后，后端在此对象中验收文件赛博档案。

2. **`formidable` (第三方库)**：
   - 不绑定框架，天生擅长大文件、多文件流式吞吐。
   - 回调中自动将数据拆解为 `fields`（普通文本）和 `files`（文件对象）。

>前端html
```js
//必须要加 enctype="multipart/form-data" ，才能上传数据,浏览器内置的不需要导入
<form action="/api/upload" method="POST" enctype="multipart/form-data">
    <input type="file" name="avatar" />
    <button type="submit">上传</button>
</form>
```
>后端node.js
```js
// 1.后端界面需要导入multer
import multer from 'multer'

// 2. 初始化multer,按照upload这个路径
const upload = multer({ dest: 'uploads/' }) 

// 3. 静态托管：让外界能通过网址访问到 uploads 文件夹里的文件
app.use('/uploads', express.static('uploads'))
app.use(express.static('.')) // 托管当前文件夹，好让浏览器能看到 html

// 4. 核心接口：接收叫 'avatar' 的文件字段
app.post('/api/upload', upload.single('avatar'), (req, res) => {

  // 后端打印看一下文件信息，file是multer修改req新增加的
  console.log(req.file) 
  
  // 输出结果，把图片的新在线链接发过去
  res.send(`<h1>上传成功！</h1><img src="/uploads/${req.file.filename}" width="200"/>`)
})
```
