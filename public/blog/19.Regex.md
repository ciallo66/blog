# 正则表达式

## 一.介绍
正则表达式（Regular Expression，常简写为 Regex/RegExp）是一门通用的**文本处理技术**。它通过一套由普通字符和特殊符号组成的“文本密码”，用来描述、匹配、提取或替换符合特定规则的字符串。

<br>

## 二.应用场景
在日常开发（如 Android 开发、后端开发、数据库管理）中，正则表达式主要用于以下三个场景：

### 1. 格式校验 (Validation)
* **定义**：检查用户输入的文本是否符合既定规则。
* **常见场景**：Android/Web 端的注册登录界面，校验用户输入的手机号（是否为11位数字）、邮箱（是否包含@和域名）、身份证号、密码强度等。
* **优势**：避免编写冗长且易错的 `if-else` 条件判断。

### 2. 文本提取 (Extraction)
* **定义**：从大段、结构复杂的文本中，精准抠出符合规则的特定数据。
* **日志分析**：在服务器几万行运行日志中，快速抓出所有的 IP 地址、报错状态码（如 404、500）。
* **网络爬虫**：从网页的 HTML 源码中，批量捞出所有的图片链接或商品 ID。

### 3. 字符串替换 (Replacement)
* **定义**：批量查找符合某种规则的文本，并将其替换为新的内容。
* **常见场景**：在代码编辑器或 DBeaver 数据库工具中，利用正则快捷键进行批量重构。例如：将文本中所有的 `user_1`... `user_99` 统一替换为 `userInfo`。

<br>

## 三.如何使用
**格式**: `^ [界定内容] $ [全局修饰符]`
> `^`表示开头，`$`表示结尾 ，`界定内容`表示规范的格式，`全局修饰符`控制检索状态，如是否忽略大小写

### (1)使用表格
| 核心检查内容 | 完全内容写法（全用 `[]` 或 `()`） | `\` 简写方式 |
| :--- | :--- | :--- |
| **纯数字** | `[0-9]` | `\d` |
| **非数字** | `[^0-9]` | `\D` |
| **字母、数字、下划线** | `[a-zA-Z0-9_]` | `\w` |
| **非字母数字下划线** | `[^a-zA-Z0-9_]` | `\W` |
| **空白字符（空格/制表符）** | `[ \t\n\r\f\v]` | `\s` |
| **非空白字符** | `[^ \t\n\r\f\v]` | `\S` |
| **单词边界位置** | (?<=[^a-zA-Z0-9_])(?=[a-zA-Z0-9_])(?<=[a-zA-Z0-9_])(?=[^a-zA-Z0-9_]) | `\b` |
| **非单词边界位置** | (?<=[a-zA-Z0-9_])(?=[a-zA-Z0-9_])(?<=[a-zA-Z0-9_])(?=[a-zA-Z0-9_]) | `\B` |
| **纯中文汉字** | `[一-龥]` | 无 | 
| **词组多选一** | `(选项A\|选项B)` | 无 |

### (2)正则表达式组件三大分类

#### 1. 字符类家族 (Character Classes)
**本质**
  必须吃掉字符。光标向后移动，在文本里占一个坑位。  
**核心符号**
- 精确单字：`a`, `b`, `c`
- 中括号范围：`[0-9]`, `[a-z]`, `[一-龥]`
- 预设简写：`\d`, `\w`, `\s`, `\D`, `\W`, `\S`


#### 2. 边界匹配器家族 (Boundary Matchers)
**本质**
  只看缝隙，不吃字符。光标原地不动，只判断当前的缝隙是不是物理边界。  
**核心符号**
- 开头结尾：`^`, `$`
- 单词边界：`\b`, `\B`
- `\b` 原形：`(?<=[^a-zA-Z0-9_])(?=[a-zA-Z0-9_])|(?<=[a-zA-Z0-9_])(?=[^a-zA-Z0-9_])`

#### 3. 断言/环视家族 (Assertions / Lookaround)
**本质**
  自定义位置探测器，不吃字符。光标原地不动，像雷达一样去偷看缝隙左边或右边符不符合你定的条件。  
**核心符号**
- 向左看必须是：`(?<=...)`
- 向左看绝对不能是：`(?<!...)`
- 向右看必须是：`(?=...)`
- 向右看绝对不能是：`(?!...)`

## 四.注意事项
1. 在正则里 `.*` 代表任意， `.`代表任意字符， `*` 代表任意长度
2. `\d{3}(\w{3})(?:\s{1}|\d{1})`  `()`可以表示创建一个抽屉,根据先后顺序使用$1可以表示`\w{3}`;如果只是想多选一，在创建括号的时候使用这种格式`(?:a|b)`即可不创建抽屉
3. 贪婪（`.*`） vs 惰性（`.*?`）假如写 a.*b，`.*`会从头到尾，把最开始的a和最后的b包在一起；如果使用`.*?`，只会把开始的a和遇见的第一个b中间的内容包裹起来
4. 字符类取反，[^a]，可以把除了a之外的所有都包含进去，相当于高级课筛选的 `.*`
