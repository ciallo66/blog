# ES6语法

### 1.Array.from( [空位一] , [空位二])
1. **空位一**:【必填：伪数组或长相怪异的对象】
2. **空位二**:【选填：挨个处理的加工函数】
    >(currentValue,index)=>{}
    
***优势***
- Array.from() 是一个纯函数，不产生多余的、需要被临时照顾的中间变量，出bug几率很低    
- 传统的for，push数组长度是不固定的，是稀疏数组，影响性能，此方法直接定义长度，为密集数组

```ts
//创建一个新的数组
const starObjects = Array.from({length:100}, (_, i) => ({        //length:100是让其预留100个空位，因为没有数据用_占位
    id: i,
    left: `${Math.random() * 100}%`,
    top: `${Math.random() * 100}%`
}));
```

### 2.类的格式
```js
class 类名 {
  
  // ==========================================
  // 1. 静态属性与静态方法（属于类本身，不需要 new 就能直接调用）
  // ==========================================
  static 静态属性名 = 静态属性值;

  static 静态方法名(参数) {
    // 静态方法的业务逻辑
  }

  // ==========================================
  // 2. 构造函数（每次 new 实例时，最先且自动执行的地方）
  // ==========================================
  constructor(参数1, 参数2) {
    // 如果有继承（extends），这里第一行必须写 super(参数);
    
    this.实例属性名1 = 参数1; // 初始化实例的属性
    this.实例属性名2 = 参数2;
  }

  // ==========================================
  // 3. 拦截器 / 存取器（用来监听或包装属性的读写）
  // ==========================================
  get 属性名() {
    // 当别人【读取】这个属性时执行的代码
    return 包装后的值;
  }

  set 属性名(新值) {
    // 当别人【修改】这个属性时执行的代码，通常用来做校验
  }

  // ==========================================
  // 4. 实例方法（最常用的地方，定义实例能干什么）
  // ==========================================
  方法名1(参数) {
    // 实例方法1的业务逻辑
  }

  方法名2(参数) {
    // 实例方法2的业务逻辑
  }

}
```

### 3.类的继承
```js
//1.创建一个新的类extends 导入需要继承的类
class SeckillGoods extends Goods

//2.在构造函数里面super导入继承的数据(name, price)
constructor(name, price, time) {
    super(name, price);
    this.time = time;
  }
```
