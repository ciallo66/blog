# Vue3新内容学习

## 1.ref vs reactive
| 类型 | 支持数据 | 读写方式 | 重新赋值 | 解构/传参 |
|------|----------|----------|----------|------------|
| ref | 基本数据,数组） | `.value` | 安全 | 安全 |
| reactive | 仅对象/数组 | 直接 `.属性` | ❌ 丢失响应 | ❌ 丢失响应 |

**使用建议**：优先用 `ref`，除非是很大的对象，并且不用从新赋值和传参

**示例**：
```js
const count = ref(0)
count.value++

const obj = reactive({ a:1 })
obj.a++
// obj = { a:2 } 错误
```

## 2.computer
- **缓存**：依赖不变，多次访问不重新计算。
- **懒执行**：依赖变化且被读取时才重算。
- **返回值**：只读的 ref 对象。
- **可写**：用 `{ get, set }` 形式。

```js
// 只读
const double = computed(() => count.value * 2)

// 可写
const full = computed({
  get: () => first.value + last.value,
  set: (v) => { /* 拆分赋值 */ }
})
```
## 3.监视
### (1)watch
- 只能监视一下数据
>1.ref 定义的数据  
>2.reactive 定义的数据  
>3.函数返回的一个值(getter函数)  
>4.一个包含上述内容的数组  

- 特点：
**懒执行**：默认只在侦听源变化时执行，不会立即运行。  
**侦听源**：可以是 ref、reactive 响应式数据、getter 函数，或多个源的数组。  
**深度监听**：监听对象/数组内部变化需开启 `{ deep: true }`,用reactive默认深度监视,且无法关闭。  
**立即执行**：配置 `{ immediate: true }` 让回调初始时立即运行一次。  
**停止监听**：watch 函数返回一个停止函数，调用即可停止。  

```js
// 单个 ref
watch(()=>count, (newVal, oldVal) => {})

// getter 函数
watch(() => state.name, (newVal, oldVal) => {})

// 多个源
watch([()=>count, ()=>state.name], ([newCount, newName], [oldCount, oldName]) => {})

// 深度 + 立即
watch(()=>count, (newVal, oldVal) => {}, { deep: true, immediate: true })
```
### (2)watchEffect
- 特点：可以自动对操作的属性进行监视，不用写监视的属性
``` js
watchEffect(()=>{
    if(a>=6 || b>=4){
        console.log('BIG!')
    }
})
//自动监视a和b
```


## 5.hooks封装
- 1.创建文件/hooks/useSum.ts (一般以  use开头+作用  小驼峰写法)
- 2.导入所需的数据方法等并暴露
``` ts
import {ref,reactive} from 'vue'

export default function(){
    let sum = ref(0)
    function add(){
        sum.value +=1
    }
    return {sum,add}
}
```
- 3.最后在用到该封装内容的地方导入，然后分解构值
``` ts
<script setup name="Man" >
import useSum from '@/hooks/useSum';
const {sum,add} = useSum()
</script>
```

## 6.toRefs和toRef
- **作用**：如果普通解构赋值就会导致解构出的数据失去响应式，外面套一个toRefs或toRef就行了

- toRef:解构一个内容
``` js
const user = reactive({ name: '张三', age: 18 })     

const nameRef = toRef(user, 'name')     //把user里的name解构出来

nameRef.value = '李四'                  //解构后仍然是ref，需要带.value
```

- toRefs:可以吧对象里的全部解构
``` js
const user = reactive({ name: '张三', age: 18 })

const { name, age } = toRefs(user)

name.value = '王五'
```

## 7.vue3路由新写法
### (1)路由配置
- 工作模式的需要导入，创建路由也需要导入
```ts
import { createRouter ,createWebHashHistory,createMemoryHistory} from "vue-router";
import Home from '@/components/Home.vue'

const router = createRouter({
    history:createMemoryHistory(),        //工作模式
    routes:[
        {
            path:'/home',
            component:Home
        },  
    ]
})
export default router

//在main.ts里
import router from './router'
const app = createApp(App)      
app.use(router)
app.mount('#app')
```

### (2)路由的props
#### 将路由收到的所有**params**参数作为props传给路由组件(只能用params参数)
``` ts
name:'home'
path:'/home/:id/:title',
component:Home
props:true

//Home组件接收,收到的数据可以直接使用
defineProps(['id','title'])            
```
#### 函数写法，可以自己决定将什么作为props给路由组件
``` ts 
name:'home'
path:'/home/:id/:title',
props(route){                   //括号的里的参数就是route，直接返回query里面的数据
    return route.query              
}
//Home组件接收
defineProps(['id','title']) 
```
#### 对象写法，直接传递对象里的数据(数据是死的，一般不用)
```
props:{
    a:'100',b:'200'
}
```
